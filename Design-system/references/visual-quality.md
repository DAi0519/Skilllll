# 视觉质量与交互规范 (Visual Quality & Interaction Guide)

## 一、设计自检方法论

设计系统预览页和实际产品页面生成后，按以下 5 步自检。每步是独立的审查视角，不跳过不合并。

### Step 0: 建立上下文

- **这是什么？** — 产品类型、页面用途、目标用户
- **用户的情绪状态？** — 压力大（财务/法律类）vs 轻松（社交/娱乐类），这决定设计的克制程度

### Step 1: 第一印象

一段话描述直觉反应。哪里突出？哪里不对？整体感受如何？直接、诚实，不含糊。

### Step 2: 视觉设计审查

| 维度 | 具体检查 |
|------|---------|
| **颜色意图性** | 每个颜色是否有目的？是否有竞争性的强调色？是否有不建立层级的装饰色？ |
| **字体层级** | 从最重要到最不重要是否有清晰的阶梯？数一数不同的字号/字重组合——数量是否合理？ |
| **阴影与边框质量** | 阴影是否清晰有层次（非模糊一片）？边框是否过于突出抢内容？ |
| **视觉重量 vs 语义重要性** | 视觉上最重的元素是否就是语义上最重要的？装饰元素是否抢了主操作的注意力？ |
| **间距与对齐** | 间距是否一致？元素是否对齐到网格？是否有过多 padding 导致内容远离应在的位置？ |
| **Icon 一致性** | 所有 icon 是否来自同一套？笔画粗细是否一致？光学尺寸是否统一？ |

每个问题的表述格式：
> **[问题名称]** — [具体事实观察]。[对用户的影响]。[可以怎么改]。

要量化：数元素数量、说出颜色值、度量相对尺寸。"有 4 种不同的背景色在争夺注意力" 优于 "颜色太多了"。

### Step 3: 界面设计审查

| 维度 | 具体检查 |
|------|---------|
| **聚焦机制** | 用户是否清楚先看哪里？是否有视觉入口？还是所有内容同等竞争注意力？ |
| **渐进式披露** | 复杂度是否逐步展示？还是一次性倾倒给用户？屏幕上有 40 个东西而只需要 5 个？ |
| **信息密度** | 密度是否适合场景？数据仪表盘可以密集，新手引导不应该密集。 |
| **预期设置** | 用户是否知道接下来会发生什么？进度是否被传达？范围是否清楚？ |
| **反馈与奖励** | 界面是否确认了用户的操作？完成的项目是否被标记？是否有前进感？ |
| **冗余** | 标签/标题/描述是否重复了用户已知的信息？能否删掉某些内容而不丢失含义？ |

### Step 4: 一致性检查

| 维度 | 具体检查 |
|------|---------|
| **模式一致** | 相似的操作是否用相同的方式处理？交互模式是否无理由地变化？ |
| **平台惯例** | 是否遵循平台规范（iOS/Android/Web）？偏离是有意的改进还是意外？ |
| **组件复用** | 是否有看起来应该是同一个组件但实际不是的元素？ |
| **视觉语言统一** | 界面是否像一个设计师做的？还是像从不同套件拼出来的？ |

### Step 5: 用户情境

- 这个设计让用户**感受到什么**？（命名情绪：压迫感、困惑、不被支持、被催促）
- 用户可能的**心理状态**是什么？（焦虑？专注？随意浏览？时间紧迫？）
- 界面是否**尊重**这个状态？还是增加了不必要的认知负担？
- **超出预期的关怀**会是什么样子？什么会让用户因体贴而惊喜？

### 问题严重度排序

1. **结构性** — 信息架构、缺失功能、错误心智模型（改变界面"是什么"）
2. **行为性** — 响应方式、流程、沟通（改变界面"感觉如何"）
3. **视觉性** — 颜色、字体、间距、阴影（改变界面"看起来如何"）

## 二、动画规范

### 动画设计原则

1. **可读优于花哨** — 任何人扫一眼文件顶部就能理解完整动画序列
2. **默认可调** — 每个影响时间或外观的值都是命名常量，随时可调
3. **数据驱动** — 重复元素用数组 + `.map()`，不复制粘贴
4. **阶段驱动** — 用单个整数 `stage` 驱动整个序列，不用散落的布尔变量
5. **弹簧优先** — 优先使用弹簧物理而非基于时长的缓动，运动更自然

### 弹簧动画预设

不同组件类型推荐不同的弹簧参数：

| 组件类型 | stiffness | damping | 效果 | 适用场景 |
|---------|-----------|---------|------|---------|
| **卡片/容器** | 300 | 30 | 平滑稳定 | 卡片入场、面板展开 |
| **弹出/徽章** | 500 | 25 | 干脆利落 | Toast、通知、点赞反馈 |
| **滑入/入场** | 350 | 28 | 平衡自然 | 列表项入场、页面过渡 |
| **微交互** | 400 | 20 | 轻快弹性 | 按钮 hover、icon 点击 |
| **拖拽回弹** | 200 | 20 | 柔软弹力 | 下拉刷新、卡片拖放 |

### 常用动效模式参数

```css
/* fadeIn — 淡入 */
opacity: 0 → 1
spring: visualDuration 0.4s, bounce 0

/* scaleUp — 缩放入场 */
scale: 0.9 → 1.0
spring: visualDuration 0.3s, bounce 0.2

/* slideIn — 滑入 */
translateY: 20px → 0
opacity: 0 → 1
spring: visualDuration 0.4s, bounce 0.1

/* hoverCard — 卡片悬浮 */
scale: 1 → 1.02
shadow: blur 8→16, offsetY 4→8
spring: visualDuration 0.2s, bounce 0.1

/* glassEffect — 玻璃效果 */
backdrop-filter: blur(16px)
opacity: 0.8
border-radius: 16px
```

### CSS 动画 Token

在设计系统中定义标准动画 Token，确保整个产品动画一致：

```css
:root {
  /* 时长 */
  --duration-instant: 100ms;
  --duration-fast: 200ms;
  --duration-normal: 300ms;
  --duration-slow: 500ms;

  /* 缓动 (非弹簧场景) */
  --ease-default: cubic-bezier(0.2, 0, 0, 1);
  --ease-in: cubic-bezier(0.4, 0, 1, 1);
  --ease-out: cubic-bezier(0, 0, 0.2, 1);
  --ease-bounce: cubic-bezier(0.34, 1.56, 0.64, 1);

  /* Reduced Motion */
  --motion-duration: var(--duration-normal);
  --motion-transform: 1;
}

@media (prefers-reduced-motion: reduce) {
  :root {
    --motion-duration: 0ms;
    --motion-transform: 0;  /* scale/translate 变化量归零 */
  }
}
```

使用方式：
```css
.card {
  transition: transform var(--motion-duration) var(--ease-default),
              box-shadow var(--motion-duration) var(--ease-default);
}
.card:hover {
  transform: scale(calc(1 + 0.02 * var(--motion-transform)));
  box-shadow: 0 calc(8px * var(--motion-transform)) 16px rgba(0,0,0,0.1);
}
```

### 动画 Storyboard 格式

为复杂多阶段动画提供标准文档格式（写在组件文件顶部或 design-system.md 中）：

```
ANIMATION STORYBOARD — [组件名]

    0ms   等待触发 (scroll into view / mount / click)
  300ms   卡片淡入, scale 0.85 → 1.0
  900ms   标题文字高亮
 1500ms   列表项滑入 (staggered 200ms)
 2100ms   CTA 按钮淡入
```

规则：
- 毫秒值右对齐，便于扫视
- 用 `→` 表示值变化 (如 `scale 0.8 → 1.5`)
- 括号标注 stagger 间隔
- 每个阶段一行，描述简短

## 三、预览页自检清单

生成 design-system.html 后，按以下清单自检：

### 布局与层级
- [ ] 侧栏纯文字、无装饰，scroll-spy 高亮当前章节
- [ ] 信息层级清晰：Section 标题 > Sub-section > 内容
- [ ] 内容区留白充足，不拥挤
- [ ] 颜色色块足够大（≥ 60×60），标注清晰（名称 + 色值）

### 精简度
- [ ] 只列项目实际使用的 Token（不堆砌完整阶梯）
- [ ] 每个 Token 标注使用场景（"用于卡片背景" 而非仅列值）
- [ ] 无冗余章节（项目不用 3D 就不列 3D 章节）

### 可交互
- [ ] 按钮 hover/active 状态可体验
- [ ] 输入框可聚焦，显示 focus 样式
- [ ] Checkbox/Toggle 可点击切换
- [ ] 卡片 hover 有阴影/浮起反馈
- [ ] Dark Mode 可切换

### 组件完整度
- [ ] 每个组件展示所有变体（primary/secondary/ghost/danger）
- [ ] 每个组件展示所有尺寸（sm/md/lg）
- [ ] 展示禁用态 (disabled)
- [ ] 展示加载态 (loading) — 如有

### App Preview
- [ ] 至少 2 个手机帧（列表页 + 详情页）
- [ ] 使用真实内容而非 Lorem ipsum
- [ ] 所有元素使用设计系统 Token（无硬编码值）
